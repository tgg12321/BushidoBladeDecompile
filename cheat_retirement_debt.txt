# Cheat retirement debt -- functions with cheats not auto-detectable by
# audit_asm_cheats.py that still need pure-C retirement per SOTN-grade
# policy (user directive 2026-05-17: "If it can be C, it should be C").
#
# Use this file when the LLM auditor flags a commit retroactively (after
# the cheat is already merged) and the agent's "retirement" mechanism is
# itself a deviation: register-asm pins to force allocation, aliasing
# barriers used for scheduling control, etc. These don't trip the
# programmatic detectors (which look at regfix.txt patterns and src/
# `__asm__` syntax) but they ARE debt per policy.
#
# Format: one entry per line -- `<funcname>  # reason / source-commit`
#
# Entries are surfaced via `dc.sh next-cheat-cleanup` alongside the
# auto-detected cheat queue, tagged `retirement_debt`. Sort behaviour
# follows the rest of the cheat queue (simpler-to-retire first).
#
# Lifecycle:
#   - Add an entry when LLM auditor flags a deviation that landed before
#     the programmatic gate was tightened, OR when a deviation is
#     subjectively a cheat but mechanically allowed.
#   - Remove the entry when the function has been retired to true pure C
#     (no inline asm, no register pins, no regfix rules).
#   - Existing pre-Tier-1-gates aliasing barriers in code6cac.c (lines
#     703, 2234, 2254, etc.) are NOT listed here — they're grandfathered
#     SOTN-bar debt and a separate sweep should track them.
