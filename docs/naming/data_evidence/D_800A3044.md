# D_800A3044 -- naming evidence

- Address: `0x800A3044`
- Proposed name: `g_spu_reg_base_ptr`
- Confidence: **high**

## Storage / access pattern

.word in .bss, holds a pointer to SPU register region.  Accesses are
volatile-u16 byte-offset reads/writes ORing with channel-control bit
masks -- the canonical SPU MMIO pattern.

## Usage in src/

src/main.c -- `cpu_side_move_dir_3`:
  spu = (void *)D_800A3044;
  *((volatile u16 *)(((s32)D_800A3044) + 0xA)) |= 0x20;

src/main.c -- `func_8008BEA4`:
  s32 reg = D_800A3044;

## Why this name

Pointer to the SPU control-register block.  +0xA OR'd with 0x20 maps to
the SPU voice-on/off control bits; storing as a runtime pointer (vs. a
compile-time constant) is how PsyQ SDK applications take the address from
`SsGetAddr` or one of the libsnd helpers at boot.
