# D_800A36AC -- naming evidence

- Address: `0x800A36AC`
- Proposed name: `g_frame_parity`
- Confidence: **high**

## Storage / access pattern

Single .word in .bss, read with `lw` and tested with `& 1`. Also used as a
scaled value:
  D_800F661A = (s16)(D_800A36AC << 2);

## Usage in src/

src/ings.c -- `func_80016E60` (display swap):
  s32 idx = D_800A36AC & 1;
  fb_base = *(&D_800A3770 + (D_800A36AC & 1));

src/code6cac_c2.c -- `func_8003DBE4`:
  colors = (s32 *)((u8 *)colors + (D_800A36AC & 1) * 24);

## Why this name

Double-buffer parity counter. The `& 1` mask used to switch between two
framebuffer/palette banks is the universal double-buffering idiom; the
`<<2` write to D_800F661A propagates the current frame index to a shared
display variable. Almost certainly a vsync-incremented frame counter.
