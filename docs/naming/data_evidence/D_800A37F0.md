# D_800A37F0 -- naming evidence

- Address: `0x800A37F0`
- Proposed name: `g_memcard_event_disconnect`
- Confidence: **high**

## Storage / access pattern

.bss word holding a BIOS event handle for memcard disconnect
(class 0xF4000001, mode 0x8000).

## Usage in src/

src/code6cac_c.c -- `gnd_close_8004939C`: extern s32 D_800A37F0;

src/code6cac_c.c -- `pad_press_control` (memcard_event_init):
  D_800A37F0 = bios_OpenEvent(0xF4000001, 0x8000, 0x2000, 0);
  bios_EnableEvent(D_800A37F0);

## Why this name

Memcard disconnect/error event handle.  Class 0xF4000001 = memcard,
mode 0x8000 = error/disconnect notification.  Another of the 8
memcard events.
