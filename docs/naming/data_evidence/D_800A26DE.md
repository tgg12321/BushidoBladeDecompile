# D_800A26DE -- naming evidence

- Address: `0x800A26DE`
- Proposed name: `g_alarm_active_sentinel`
- Confidence: **high**

## Storage / access pattern

.bss byte.  The sentinel value 0x7F that irq_ProcessPending writes to
this address indicates 'alarm armed'.  Tested as `== 0x7F` and `== 0`
for branch decisions.

## Usage in src/

src/ings2.c -- `bios_FileRead`: extern u8 D_800A26DE;

src/ings2.c -- `irq_ProcessPending`:
  if (D_800A26DE == 0x7F) { ... }
  else if (D_800A26DE == 0) { ... }

## Why this name

Alarm-active sentinel byte.  0x7F = alarm armed and pending;
0 = alarm inactive.  Written by irq_ProcessPending at the tail of
the critical-section block (the sentinel write I documented in batch 6).
