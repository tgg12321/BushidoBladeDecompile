## Why this name

* `g_special_cam_command_table` -- captures (a) the SpecialCam
  subsystem the function feeds into via the body's `SpecialCam +
  D_80101E60 * 8` access, and (b) the table's role as a
  command-descriptor pool (callers pass small command codes which
  index this table).
* Confidence **high** because:
  1. The table is touched by exactly one function (`func_80037110`),
     and that function does nothing but consume table rows -- no
     ambiguity about what the data is used for.
  2. The 8-byte stride is deterministically derivable from the asm.
  3. The per-row field layout is directly readable from the code.
