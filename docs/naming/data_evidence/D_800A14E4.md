# D_800A14E4 -- naming evidence

- Address: `0x800A14E4`
- Proposed name: `g_irq_alarm_handle`
- Confidence: **high**

## Storage / access pattern

Volatile .bss word.  Set to -1 (sentinel for 'no alarm') via volatile
write -- the volatile suggests an IRQ handler is the consumer.

## Usage in src/

src/system.c -- `tslTm2LoadImage_2`: extern s32 D_800A14E4;

src/system.c -- `saEft00Add`:
  *(volatile s32 *)&D_800A14E4 = -1;

## Why this name

IRQ alarm handle; -1 means no alarm armed.  Volatile because the
alarm IRQ handler reads it concurrently with the main thread's writes.
