## How to use this catalog

These names are **NOT** applied to `named_syms.txt`.  They're proposals in case a future researcher needs a searchable handle for any of these globals.  The mechanical-name format (`g_<kind>_<addr>`) is intentional: it tells you the access pattern at a glance.

To upgrade a mechanical name to a semantic name:

1. Pick a global from this catalog.
2. Read every callsite (grep `D_<addr>` in `src/`).
3. Infer the role (paired counter? state-machine cursor? cache?).
4. Write an evidence file in `data_evidence/D_<addr>.md`.
5. Add to `named_syms.txt` if confidence is high.

Most globals in the 1-2 refs band don't reward this effort -- they're
narrowly-scoped helpers whose meaning only emerges when the enclosing
function is fully decompiled.
