### Layout matches a PSX GPU coordinate-pack command (rank=medium)

The bit-field shape -- two 10-bit coordinate fields plus two 1-bit
flags packed into a single u32 -- is characteristic of PSX GPU
register commands. The PSX GPU's `GP0(E5)` "Drawing Offset" command
packs X (bits 0..10) and Y (bits 11..21) signed offsets; the GPU
`GP1(05)` "Display Mode" command packs HRES/VRES/area flags.

Our output is 22 bits wide -- specifically 10 (X-ish) + 10 (Y-ish)
+ 2 (boolean flags). The closest match is the **packed display /
draw coordinate format** with two short coordinate values plus two
mode flags.

The `disp_*` prefix is justified by the consumer's role:
`func_80077984` ends with `disp_SetFramebufferMode(1, 0, 0, 0)`.
