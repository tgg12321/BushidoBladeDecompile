## Evidence

### Trivial wrapper around `cdrom_DmaToRam` (rank=high)

The function inverts the return value of `cdrom_DmaToRam` (a PsyQ /
BB2 helper that returns the active DMA target address, or 0 when no
transfer is in progress). So the wrapper returns:

* `1` when no DMA-to-RAM transfer is active.
* `0` when a transfer is in progress.

This is the "**is the CD DMA idle right now?**" predicate -- the
caller-friendly "true means ready" inverse of the underlying
helper's "true means busy / address" return convention.
