# func_80044498 -- naming evidence

- Address: `0x80044498`
- Source: `src/text1a_c.c`

- Confidence: **medium**

## Body

```c
void func_80044498(void) {
    s32 i = 0x13;
    s16 *p = &D_8010367E;
    for (; i >= 0; i--) {
        *p-- = 0;
    }
}
```

## Evidence

**Clears 20 (= 0x13 + 1) s16 entries** at addresses `D_8010367E` down through `D_80103658`. The walk-backwards iteration with `*p-- = 0` is GCC 2.7.2's idiomatic short-form for `memset`-of-aligned-s16.

Note that `D_80103658` is the partner-array bound of `g_tpage_slot_count[]` (see [`D_80103658.md`](../data_evidence/D_80103658.md)). This function clears 20 entries of the slot-count array -- the **TPAGE-slot-pool reset** routine.

## Why this name

Clears the entire 20-slot TPAGE pool by zeroing every entry in `g_tpage_slot_count[0..19]`. Confidence medium because the array identification is solid (the address matches the data evidence file exactly), but the function name could be `tpage_pool_reset_*` instead.
