## Evidence

### Sets a sequence-state record (rank=high)

The function writes 6 globals at consecutive addresses (`D_800A9CF8`
through `D_800A9D04`) -- a 14-byte record:

| Offset           | Width | Source     | Meaning                           |
|------------------|-------|------------|-----------------------------------|
| `D_800A9CF8` +0  | s16   | `a1`       | Sequence ID / key                  |
| `D_800A9CFA` +2  | u16   | `*a0` (input head) | Header word from input array |
| `D_800A9CFC` +4  | s32   | `stage_GetId()` | Current stage ID            |
| `D_800A9CFE` +8  | s16   | switch on stage | Per-stage offset multiplier |
| `D_800A9D00` +0xC| s32   | `a0+1`     | Input cursor (post-header)        |
| `D_800A9D04` +0x10| s32  | `a2`       | Data block ptr                    |

It then returns `a2 + D_800A9CFE * 104`, where `D_800A9CFE` is a
per-stage offset (0x11 for stage 7, 2 for stage 4, 0xA for stage
0x12, 0 for everything else). 104 = `0x68` -- the same stride seen
in the `+0x112C / 0x68` pointer chain inside
[`func_80041430`](func_80041430.md) (player_rob_relocate).

So the function:

1. Stashes 6 fields about the sequence's input + stage into a
   global state record.
2. Returns `data_base + per_stage_multiplier * 0x68` -- a pointer
   into the data block at a stage-specific offset.

This is the **"set up the sequence state for stage N and return the
stage-specific data pointer"** pattern.

### Calls `stage_GetId()` (rank=medium)

`stage_GetId()` is already-named. Its return value drives a

### The 0x68-byte stride matches `player_rob_relocate` (rank=medium)

per-stage / per-character record table.
