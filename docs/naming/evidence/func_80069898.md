## Body shape

The function takes a `GameObj *arg0`, a `u16 *arg1` (4-halfword
rectangle), and an `arg2` (ordering-table slot index). It uses
`arg0->field_18` as a prim-list write pointer and emits **three
consecutive 16-byte PSX tile primitives**, each:

1. `initTile(p)` -- standard PSX `addPrim` SET_TILE init.
2. Write RGB triplet at `p[4..6]` -- 0x80/0x80/0x80 for tile 1, then
   0x80/0x80/0x80 for tile 2, then 0x40/0x40/0x40 for tile 3
   (each lighter than the previous).
3. Copy `arg1[0..3]` into the tile's x/y/w/h fields, with `y--` for
   tile 2 and `y -= 2` for tile 3.
4. `gpu_SetSemiTransp(p, mode)` -- mode 0 for first tile, mode 1 for
   tiles 2-3.
5. `ot_Link(D_800A374C + arg2, p)` -- attach to the OT chain.

Then `arg0->field_18` is advanced past the 3 tiles for the next
caller.

## Evidence

### Three-tile stacked alpha-blend shadow (rank=high)

The pattern is unambiguous:

* 3 identical-rectangle tiles stacked vertically by 0, -1, -2 pixels
  with decreasing color intensity (`0x80, 0x80, 0x40`).
* Each subsequent tile uses semi-transparent mode (mode 1 = 50%
  alpha vs mode 0 for the base tile).
* All three link into the SAME ordering-table slot (`+arg2`).

This is the **canonical 3-tile soft-shadow / bloom effect**: render
the base shape, then overlay 2 progressively-dimmer alpha-blended
copies above it for a soft glow. Common in PSX-era HUD elements
(damage flash, health-low warning, victory ribbon).

### Uses already-named GPU helpers (rank=medium)

`initTile`, `gpu_SetSemiTransp`, `ot_Link` are all already-named
PSX SDK / BB2-renamed helpers. Their presence anchors this function
as a `draw_*` / OT-emitter rather than something else.

### `GameObj.field_18` as prim-buffer cursor (rank=medium)

The struct field `field_18` (s32) is used as a "next prim write
pointer". This is the same convention used in
`text1a_c.c`'s prim-buffer code via
[`prim_buffer_open_slot_80044010`](func_80044010.md). The
relationship suggests `GameObj.field_18` is one of the 12 slots
managed by that pool.

## Why this name

* `draw_shadow_3tile_stack_<addr>` -- captures (a) the draw-call
  role, (b) the 3-tile structure, (c) the shadow / soft-glow
  effect indicated by the RGB-progression and semi-transp modes.
* Confidence **medium-high** because the 3-tile pattern with
  fading RGB + alpha is a forced reading; the only uncertainty is
  whether "shadow" or "glow" is the better label (both emit the
  same code shape).
