# func_80085114 -- naming evidence

- Address: `0x80085114`
- Source: `asm/funcs/func_80085114.s`
- Proposed name: `seq_channel_state_init_80085114`
- Confidence: **medium**

## Body

Indexes `D_80106F28[bank][channel*0x58]` (channel = a1, bank = a0 sign-extended).
Writes `1 -> ch[0x20]`, `0 -> ch[0x21]`, then masks five bit-clear ops on `ch[0x98]`
(masks -0x101, -9, -3, -5, -0x201). Copies `ch[4]` into `ch[0]`, writes `1 -> ch[0x14]`,
and OR-sets bit 1 in `ch[0x98]`.

## Evidence

Initializes per-channel SEQ/MIDI playback state in the D_80106F28 channel table. The bit clears at 0x98 are exactly the play-state flags; the ch[4]->ch[0] copy is the read-cursor reset.

## Why this name

Standard SEQ channel state-init. Each entry of `D_80106F28` looks like a bank pointer; the inner stride 0x58 matches a per-channel control block.
