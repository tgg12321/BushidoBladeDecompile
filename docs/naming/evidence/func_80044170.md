## Evidence

Varargs function (PSX SDK uses the &args+4 pattern for varargs). Takes a base buffer + count + a list of `(entry_index)` integers; for each entry-index, looks up `(cur_off, size)` from the table at base, copies `size` bytes via `func_800520B8` (a memcpy), and records the new offset in the slot table. Output: a compacted entry table with rebuilt offsets.

## Why this name

Builds a packed entry-table at `a0` from a subset of entries selected by index. Used for 'extract these specific entries into a smaller compact representation' -- e.g., extracting a few sprite-pages from a master TPAGE table.
