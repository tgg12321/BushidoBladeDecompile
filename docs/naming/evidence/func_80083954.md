# func_80083954 -- naming evidence

- Address: `0x80083954`
- Source: `asm/funcs/func_80083954.s`
- Proposed name: `kernel_alarm_cancel_80083954`
- Confidence: **medium**

## Body

Branches out early if `D_800A26D0 == 0` (no alarm armed). Otherwise enters a
critical section (`EnterCriticalSection`), clears `D_800A26DD`, then conditionally:
  - if `D_800A26DC` set: `irq_SetAlarm(0)` + clear D_800A26DC
  - else if `D_800A26DE` clear: `irq_EnableInterrupts(0, D_800A26D8)` + clear D_800A26D8
  - else: `irq_EnableInterrupts(0, 0)`
ExitCriticalSection, then stores `D_800A26DE = 0x7F` (sentinel).

## Evidence

Classic PsyQ alarm-cancellation pattern: critical-section wrap, conditional alarm reset, conditional IRQ re-enable, sentinel write. Mirrors the cancel-half of `cdrom_Init`/alarm-arm in `LIBSPU`.

## Why this name

Cancels (or disarms) the pending alarm/IRQ pair installed by a sibling arm function. The 0x7F sentinel write is the 'now disarmed' marker.
