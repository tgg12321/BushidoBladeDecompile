## Needs owner attention (NOT auto-edited â€” semantic conflicts, not mechanical drift)

These are cases where the doc's *behavioral description* is contradicted by a
`named_syms.txt` note/rename. Because an auto-generated name can itself be
wrong, and I was scoped to read-only for `src/`, I did not rewrite the
behavioral prose â€” the owner should adjudicate:

- **ai.md** `D_800A36A4` is described throughout as the **stance / mode-group
  selector**, but `named_syms.txt` names 0x800A36A4 `g_stage_id_current`
  ("current stage ID passed to game_StageCleanup"). One of the two is wrong.
- **ai.md** `D_8008D538[stance]` is described as a **per-stance waza-enable
  bitmap**, but 0x8008D538 is `g_char_class_id_table` (per-character class ID,
  indexed by char_id). Conflicting semantics.
- **combat.md** `action_check_defense` (0x80086130) â€” `named_syms.txt`
  explicitly supersedes this as `char_store_pos_pair_80086130` and states it is
  **not** a defense check ("stores paired values into 16-byte slot records").
  The doc's "combat block/defense check" description should be revised.
- **gpu_pipeline.md / motion.md** the `calc_loc_mat_fw*` family is documented as
  per-bone local-matrix builders, but `named_syms.txt` flags `calc_loc_mat_fw`
  (0x8002AB08) MISNAMED = a GPU DMA scheduler, and `calc_loc_mat_fw_8004A940` =
  a u16 stream decoder. If those MISNAMED notes are correct, the motion/GPU
  matrix-pipeline prose is wrong.
- **gpu_pipeline.md** the four `Vu0SetLightColMatrix*` functions are flagged
  MISNAMED in `named_syms.txt` (wrappers of `cpu_side_move_dir_4` /
  `marionation_Exec` / `cdrom_DmaChain` / `saEft01Init`), contradicting the
  "light-color matrix setup" gloss.
- **replay.md** `D_8008EC38` is described as a parallel camera-length table, but
  0x8008EC38 is `g_event_class_lookup_table` ("PSX BIOS event class IDs").
- **file_io.md / psyq_usage.md** `ang_hosei` is glossed as a BIOS file-lseek
  wrapper, but `named_syms.txt` flags `ang_hosei_80056FE8` MISNAMED = a
  character display-offset lookup (neither rotation nor file-seek).
- **`Judge` vs `g_sin_table`** (0x800973FC): docs use `Judge[]` for the sin/cos
  LUT; `named_syms.txt` now carries the canonical alias `g_sin_table`. `Judge`
  is still the live symbol in `src/*.c`, so this is a dual-name, not a break â€”
  docs could note the canonical alias but no fix is required.
