# Evidence bank — gpu_SetDispMask

- Audit diagnosis (regressions.md): Pointer-derived address cheat: `u8 *p = &g_gpu_debug_level; bb2_memset(p + 0x6A, -1, 0x14)` uses g_gpu_debug_level (0x8009BE76) as the base to compute g_gpu_disp_env (0x8009BEE0 = 0x8009BE76 + 0x6A), which IS a separately-named global in symbol_addrs.txt. The pointer `p` exists solely to keep &g_gpu_debug_level live in $s1 (callee-save) so the compiler emits `addiu $a0,$s1,0x6A` (one instruction) for the memset arg rather than a fresh lui+addiu for g_gpu_disp_env (two instructions). A clean redo must write `bb2_memset(&g_gpu_disp_env, -1, 0x14)` (or equivalent cast) and accept the changed register allocation, then find a legitimate C form that also produces the target's single-instruction memset-arg computation.
