# Each line: <func>  <reason>  <evidence>

# These functions use instructions or patterns that GCC 2.7.2 + ASPSX cannot

# Reason codes (also auto-detected by classify_func.detect_permanent_blockers):
#   break_instruction       non-GCC-div-guard break <code>; GCC's normal
#                           signed division emits break 7 / break 6 and those
#                           do NOT belong here.
#   handwritten_overflow_op add/sub/addi (overflow-trap variants); GCC emits
#                           addu/subu/addiu by default. Requires inline asm.
#   sp_manipulation         direct $sp swap or context restore (not
#                           prologue/epilogue). C has no way to take/set the
#                           stack pointer.
#   no_return_fragment      no `jr $ra` in body; orphaned code or fall-through
#                           data-as-code that must remain in .text verbatim.
#   data_as_code            instructions are actually a constant table accessed
#                           via mid-function alabels; semantically data, not code.
#   bios_jumptable_call     BIOS/kernel helper with inline handler/data tail
#                           that cannot be represented as one C function.
#   cop0_op                 mtc0/mfc0/ctc0/cfc0/rfe/tlbXX — system control
#                           coprocessor. No C primitive; requires inline asm.
#
# To remove an entry: only do so if you've actually decompiled the function
# to pure C and it builds. Don't delete just because a future GCC has the
# right builtin -- our toolchain is pinned to 2.7.2.
